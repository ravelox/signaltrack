# Design System Notes

## What this pass improves
- consistent spacing and control styling
- one place for loading, empty, and error states
- repeatable form field patterns
- repeatable list/table patterns
- easier visual refinement later

## New core primitives
- Button
- Field
- Input
- Textarea
- SectionHeader
- EmptyState
- ErrorPanel
- TableSkeleton
- DetailSkeleton
- DataTable

## Recommended next move
Use these primitives to implement:
1. real form validation states
2. row selection and bulk actions if needed
3. responsive table alternatives for smaller screens
4. admin/audit screens
